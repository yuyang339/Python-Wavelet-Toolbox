DiscreteWavelets Toolbox    

Programming Conventions

Here is a list of programming conventions used to create the DiscreteWavelets Toolbox. Hopefully, they will be helpful as you use the toolbox.

Global Variables

The DiscreteWavelets Toolbox makes use of five global variables. All are initialized by the m-file startup.m.

DAUBLENGTH
this variable keeps track of the length of the current Daubechies filter (this value is 0 if the filter is not defined). The Daub function calls an auxiliary function Daubfun to solve the quadratic system that gives the Daubechies filter. DAUBLENGTH is passed by Daub to Daubfun to accomplish this task.

COIFLENGTH
this variable keeps track of the length of the current Coiflet filter (this value is 0 if the filter is not defined). The Coif function calls an auxiliary function Coiffun to solve the quadratic system that gives the Daubechies filter. COIFLENGTH is passed by Coif to Coiffun to accomplish this task.

DWDATAPATH
this variable gives the path where all data files included with the DiscreteWavelets Toolbox can be found. The m-file startup.m defines this variable by searching the output of pathdef so in order for the function to be successful, you must have added the path (and all subpaths) where you installed the toolbox using the Matlab addpath command or via the GUI under File.

DWIMAGESATH
this variable gives the path where all image files included with the DiscreteWavelets Toolbox can be found. The m-file startup.m defines this variable by searching the output of pathdef so in order for the function to be successful, you must have added the path (and all subpaths) where you installed the toolbox using the Matlab addpath command or via the GUI under File.

DWSOUNDSPATH
this variable gives the path where all sound files included with the DiscreteWavelets Toolbox can be found. The m-file startup.m defines this variable by searching the output of pathdef so in order for the function to be successful, you must have added the path (and all subpaths) where you installed the toolbox using the Matlab addpath command or via the GUI under File.

Function Naming

All functions in the DiscreteWavelets Toolbox, with the exception of nCE, start with a capital letter. In the case where two or more words are used to describe the function, the first letter of each word is capitalized (e.g., WaveletDensityPlot).

Output

Any function that outputs a vector (or vectors) returns the vector as a column vector. Functions that need vectors as input will accept either row vectors or column vectors.

Optional Arguments

Several functions in the DiscreteWavelets Toolbox can be supplied optional arguments. These arguments must always be provided in pairs. The convention is 'Option','Value'. In some cases, 'Value' is a string, but in other cases, it is a scalar, vector, or matrix. If 'Value' is a string, it must be entered as such (e.g., 'Hello World' as opposed to Hello World). Some functions can be provided several optional arguments - the order the arguments are provided does not matter (the functions parse the optional variables provided and replace default values with input values), but the input must follow the 'Option','Value' convention.

As an example, consider the function HuffmanTree. There are three required arguments for this function so the basic call is

HuffmanTree(u,f,c)

There are several optional arguments. Included in this list is 'NodeSize', 'Title', and 'NodeColor'. Here are some valid calls using the optional arguments:

HuffmanTree(u,f,c,'NodeColor',[1 0 0],'NodeSize',20,'Title','A Huffman Tree');
HuffmanTree(u,f,c,'Title','A Huffman Tree','NodeSize',20);
HuffmanTree(u,f,c,'Title','A Huffman Tree','NodeSize');

In the last call, the routine simply ignores 'NodeSize'.

Error Handling

The DiscreteWavelets Toolbox does not utilize the Matlab error function. Instead, the functions in the toolbox attempt to locate invalid arguments and return a message and an output. In many cases, several commands are run as a script m-file. If the first command is given an invalid input, then the output is designed so that subsequent commands that require it will nevertheless successfully execute. The user should be able to figure out from the messages the problem with the invalid argument. No claim is made that this strategy will work with every combination of commands, but it does work well for common combinations of functions from the toolbox.

The functions in the DiscreteWavelets Toolbox DOES NOT check if the user has provided the required inputs for a function. If the incorrect number of inputs are given, then the function will invoke the Matlab error function.
